# Pricing Page Wireframe

Last updated: June 11, 2026

Page: `workcv.co.uk/pricing`

Purpose:

- blueprint for pricing-page structure before implementation
- aligned to the pricing copy and validated competitor-pricing logic

## Desktop layout summary

1. Sticky header
2. Hero
3. Pricing summary card
4. Competitor comparison block
5. Fit / why this model works block
6. What you get block
7. Comparison-links block
8. FAQ block
9. Final CTA block
10. Footer

## Mobile layout summary

Same section order, fully stacked.

Most important adjustment:

- competitor comparison must stay readable on mobile
- use stacked cards or horizontally scrollable comparison treatment if needed

## 1. Sticky Header

### Layout

- same header system as homepage

## 2. Hero

### Layout

- centered copy block
- H1
- subheading
- CTA row

### Content hierarchy

- eyebrow
- H1: price clarity
- subheading: no subscription, no auto-renewal
- primary CTA
- secondary CTA

### Notes

- pricing page hero should feel direct and more commercial than the homepage hero

## 3. Pricing Summary Card

### Layout

- centered dominant card
- price figure large
- one-time label beneath or beside price
- included points list
- primary CTA inside or directly below card

### Card structure

- card title: `WorkCV`
- `GBP 4.99`
- `one-time payment`
- 4 to 5 support points
- small reassurance line

### Notes

- this is the anchor of the page
- make it visually stronger than surrounding sections

## 4. Competitor Comparison Block

### Layout

- section label
- H2
- short intro paragraph
- comparison note
- comparison table or card system
- CTA below comparison

### Desktop options

Option A:

- full comparison table with 4 columns

Option B:

- stacked competitor cards plus a highlighted WorkCV card

### Mobile options

Option A:

- vertical stacked cards

Option B:

- row-by-row compact comparison blocks

### Required comparison fields

- entry price
- renewal price
- billing cadence
- auto-renews
- cancellation needed
- displayed currency

### Notes

- do not force a dense spreadsheet on mobile
- clarity is more important than table purity

## 5. Fit Section

### Layout

- section label
- H2
- explanatory paragraph
- 3 to 4 supporting bullets/cards

### Purpose

- shift from price difference to product-fit difference
- explain why the one-time model makes sense for one-CV users

## 6. What You Get Section

### Layout

- section label
- H2
- short paragraph
- 4-point grid or list

### Content focus

- guided builder
- clean UK templates
- final PDF download
- saved login-based CV access later

### Notes

- keep this narrower than a typical SaaS feature dump

## 7. Comparison Links Block

### Layout

- section label
- H2
- 3 comparison cards or link cards

### Links

- MyPerfectCV alternative
- LiveCareer alternative UK
- CVMaker UK alternative

### Notes

- this section should look like a natural next step, not an SEO dump

## 8. FAQ Block

### Layout

- section label
- H2
- accordion list

### FAQ count

- 6 to 7 items

### Notes

- spacing should be generous
- answers should remain compact

## 9. Final CTA Block

### Layout

- strong band with centered content
- H2
- short body
- primary CTA
- optional secondary CTA

### Purpose

- close the page with one clean action
- avoid sending the user back into analysis paralysis

## 10. Footer

### Layout

- same footer system as homepage

## Responsive behavior

### Tablet

- pricing card stays centered
- comparison section may reduce from table to card layout

### Mobile

- price card first and highly visible
- comparison content stacked and simplified
- CTA kept close to the pricing summary and comparison block

## Implementation priorities

If time becomes tight, preserve these sections first:

1. Hero
2. Pricing summary card
3. Comparison block
4. Final CTA

Those sections carry the entire commercial purpose of the page.
