# Visual Design System

Last updated: June 11, 2026

Reference direction:

- Dwight-inspired structure and clarity
- cleaner and more commercial for a one-product CV offer
- not enterprise
- not startup-purple SaaS
- not generic recruitment-site blue gradient fluff

## Design Goal

The design should make WorkCV feel:

- clear
- premium enough to trust
- simple enough to understand quickly
- modern but not loud
- UK-appropriate rather than Silicon Valley styled

The page should feel like a focused commercial product, not a content farm.

## Visual Principles

1. The price model should be visible everywhere important.
2. Sections should feel spacious, but copy should stay tight.
3. Typography should create contrast without feeling decorative.
4. Trust comes from clarity and restraint, not badges everywhere.
5. Comparisons and CTAs should stand out more than background effects.

## Typography

### Display font

Use:

- `Fraunces`

Role:

- H1
- key H2 headings
- large price figures if needed

Why:

- gives editorial weight
- feels more distinctive than default SaaS fonts
- suits the UK-commercial tone better than a pure geometric display

### Body font

Use:

- `DM Sans`

Role:

- body copy
- nav
- UI labels
- buttons
- FAQ text

Why:

- clean and readable
- pairs well with Fraunces
- less bland than system UI alone

## Colour System

### Core palette

- Background: `#F8F7F3`
- Surface: `#FFFFFF`
- Primary navy: `#0F2942`
- Navy hover: `#1A3D5C`
- Accent gold: `#D4A843`
- Gold tint: `#F5E6C0`
- Text primary: `#1A1A1A`
- Text muted: `#6B7280`
- Border: `#E5E2DB`
- Border strong: `#C9C5BC`
- Success green: `#2D7D52`
- Soft red tint for competitor negatives: `#FFF5F5`
- Soft green tint for WorkCV positives: `#F0FDF4`

### Usage rules

- navy is the main trust colour
- gold is an accent, not a background colour for whole sections
- large background areas should stay warm and quiet
- CTA priority comes from contrast, not neon colours

## Layout System

### Container

- max width: `1120px`
- side padding: `24px`

### Section spacing

- desktop vertical section padding: `88px to 104px`
- mobile: reduced but still airy

### Grid approach

- use clean two-column layouts where useful
- comparison sections should prioritise scanability over symmetry
- hero can use text left and product visual right

## Shape Language

- border radius small: `6px`
- medium: `12px`
- large: `20px`
- pill: `9999px`

Use rounded corners consistently, but avoid over-rounding every element.

## Navigation Style

Inspired by Dwight:

- logo left
- key links center/right
- one secondary CTA
- one primary CTA

Recommended nav items:

- Templates
- Pricing
- FAQ
- `See example` secondary CTA
- `Build my CV` primary CTA

Style:

- sticky header
- subtle backdrop blur
- white or slightly translucent surface
- thin bottom border on scroll state

## Hero Style

### Background

- warm off-white base
- subtle grid or line texture only if it stays quiet
- avoid heavy gradients

### H1

- large
- dark navy
- short enough to scan in one breath

### Supporting content

- 2 to 3 bullets max
- compact CTA row
- right-side product mockup or CV preview

### Product visual

- slightly elevated card stack or builder preview
- no exaggerated 3D effects
- no overcomplicated dashboard imagery

## Buttons

### Primary button

- navy background
- white text
- medium rounded corners
- subtle lift on hover

### Secondary button

- transparent or white background
- navy text
- strong border
- quieter than primary CTA but still visible

## Comparison Section Style

This is the visual centerpiece after the hero.

### Typical CV builder side

- soft red or warm neutral tint
- clearer negative markers
- keep it factual, not sensational

### WorkCV side

- soft green or white with stronger navy heading
- make the positive path feel calmer and cleaner

### Heading

- large and blunt
- must read like a commercial truth, not an abstract brand line

## Pricing Card Style

- centered or dominant within its section
- strong navy border or subtle shadow
- large price figure in display font
- clear `one-time payment` label
- short benefits list

The card should look like a confident product offer, not a pricing table from a B2B app.

## FAQ Style

- simple accordion
- thin borders
- roomy click targets
- no visual clutter

The FAQ should feel like reassurance, not filler.

## Motion Guidelines

Keep motion minimal and purposeful:

- subtle upward hover on cards/buttons
- gentle fade/slide on section reveal if used
- no constant floating, pulsing, or animated gradients

## What To Avoid

- purple-heavy SaaS palettes
- dark mode by default
- oversized gradient blobs everywhere
- generic recruitment stock visuals
- excessive icon rows
- overexplaining product mechanics in the design itself
- crowded comparison tables

## Overall Impression Target

A user should feel:

- this looks cleaner than most CV-builder sites
- this is easier to trust
- this is simpler to buy
- this is focused on getting the CV done, not trapping me in a platform
