Confirmed inputs:

- public domain: `workcv.co.uk`
- launch path: separate domain
- editor access: login required when the user enters the editor
- UK price: `GBP 4.99`

## V1 principle

This first version should not try to look like the biggest CV platform in the UK.

It should behave like the clearest one:

- simple price
- UK-correct CV structure
- fast completion
- no subscription
- direct route from landing page to builder

## CTA system

Use one clear CTA hierarchy across the site.

### Primary CTA

- `Build my CV`
- or `Build my CV for GBP 4.99` where price clarity matters

Primary CTA destination:

- editor entry flow

### Secondary CTA

Use only where needed:

- `See pricing`
- `See templates`
- `See an example`

Secondary CTA destinations:

- pricing page
- template page
- selected example page

### CTA rules

- no more than one primary CTA per section
- pricing pages can repeat the same primary CTA
- avoid splitting attention between too many route options
- if a page is comparison-led, the CTA should still resolve to the builder or pricing page

## Shared FAQ themes

Use these across V1 pages where relevant:

- How much does WorkCV cost?
- Is WorkCV a subscription?
- Do I need to log in before I can use the editor?
- Is the format right for UK jobs?
- Can I edit my CV later?
- Does WorkCV work for students or users with no experience?

## Shared content rules

Every page must follow these rules:

- use UK spelling
- no US `resume` framing unless in clarifying comparison copy
- no date of birth, marital status, or nationality in example CV structures
- no unsupported trust claims
- no stale competitor pricing
- no decorative fluff that weakens the commercial message

## What this first version is trying to learn

1. Does one-time pricing convert in the UK as clearly as expected?
2. Do early-career pages produce stronger builder intent than broader generic content?
3. Do comparison pages pull qualified traffic into pricing and editor routes?
4. Is the login-required editor flow acceptable if the marketing promise stays clear?
